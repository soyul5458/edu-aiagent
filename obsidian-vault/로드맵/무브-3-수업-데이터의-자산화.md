# 무브 3 — 수업 데이터의 자산화

## 왜

- 수업은 소멸성 매출이지만, 수강생들이 남기는 아이디어 데이터는 **유일하게 복리로 쌓이는 자산**이다: 미래의 아이디어 공유 대시보드, "수강생 N명이 만든 아이디어 N개"라는 마케팅 증거, 다음 강의 커리큘럼 개선 재료.
- 핵심 제약: **공유 동의는 수업 전에만 자연스럽게 받을 수 있다.** 지나고 나서 받으려면 개별 연락 비용이 들고 회수율도 낮다. 이번 기수를 놓치면 그 데이터는 영영 대시보드에 못 쓴다.
- 저장 구조는 이미 올바르다 — DB가 원본, 옵시디언은 지식 레이어 ([[시스템-아이디어-데이터-흐름]]). 지금 필요한 건 대시보드 "구축"이 아니라 ① 동의 ② 백업 습관 ③ 지식화 습관이다.

## 정확한 단계

1. 수업 안내문/첫 슬라이드에 동의 문구 1줄 추가:
   "실습에서 만든 아이디어는 수업 참가자 간 공유되며, 이후 익명화된 형태로 우수 사례 소개에 활용될 수 있습니다."
2. 수업 종료 직후 (5분 루틴):
   - `data/` 폴더를 날짜 이름으로 복사 (백업)
   - 관리자 페이지 → [옵시디언 내보내기] 클릭 → 볼트 `아이디어/`에 노트 생성
3. 기수가 끝날 때마다 볼트에서 아이디어 노트를 훑고 "다음 수업 개선점" 노트 1개 작성 (예: 어떤 카테고리에 몰리는가, 어디서 막히는가)
4. 대시보드는 데이터 3기수 분량이 쌓인 뒤에 설계 ([[중단-3-실사용-전-기능-개발]])

## 완료 기준 (done)

- [ ] 동의 문구가 수업 자료에 들어가 있음
- [ ] 수업 후 백업 폴더 + 옵시디언 노트가 존재함
- [ ] 기수 종료 후 개선점 노트 1개가 볼트에 있음

## 약한 모델에게 시킬 프롬프트

```
obsidian-vault/아이디어/ 폴더의 노트들을 전부 읽고 다음을 정리한 노트를
obsidian-vault/로드맵/기수-리뷰-YYYY-MM.md 로 만들어줘:
1. 카테고리별 아이디어 수와 눈에 띄는 쏠림
2. 수강생들이 공통으로 원했지만 MVP에서 제외된 기능 톱5 (원본 기획서 섹션 근거 인용)
3. 위 근거로 다음 수업에서 예시로 쓸 만한 아이디어 3개 추천
각 항목에 근거 노트를 [[링크]]로 달아라. 요약하지 말고 원자화된 근거를 남겨라.
```
