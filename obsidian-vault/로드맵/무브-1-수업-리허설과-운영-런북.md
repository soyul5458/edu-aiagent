# 무브 1 — 수업 리허설과 운영 런북

## 왜 (기대수익이 가장 큰 이유)

- 이 사업의 가치는 **수업 당일 1~2시간**에 전부 실현된다. 그 시간에 플랫폼이 멈추면 과정 전체의 신뢰가 무너지고, 초보 수강생은 "역시 어렵다"는 거부감만 안고 돌아간다.
- 실패 비용(기수 전체의 신뢰 + 재수강/소개 매출)과 준비 비용(2시간)의 비대칭이 극단적이다.
- 이미 개발 중에 **실제로 관찰된** 실패 모드가 3개다: ① 구버전 서버가 포트를 점유한 채 살아남음 ② API 키 복사 실수(fw- vs fw_) ③ 동시 요청 시 429. 관찰된 실패는 반드시 재발한다 — 리허설 없이 수업에 들어가는 것은 도박.

## 정확한 단계

1. 수업 24시간 전: 코드 동결 선언 ([[중단-2-수업-직전-시스템-변경]])
2. `kill $(lsof -t -iTCP:3000 -sTCP:LISTEN)` → `npm run build && npm start -- -H 0.0.0.0`
3. 강사 IP 확인: `ipconfig getifaddr en0` → 휴대폰(같은 와이파이)에서 `http://IP:3000` 접속
4. 더미 계정 3개로 **동시에** 정제 실행(노트북+폰+태블릿) → 전부 성공하거나 429 안내가 뜨는지 확인
5. Fireworks 대시보드에서 크레딧 잔액·rate limit 확인 (수강생 수 × 2회 정제 기준)
6. "수업 런북" 1페이지 작성: 시작 명령 / IP 확인법 / 흔한 오류 3가지와 강사 대응 멘트 / 플랜 B(플랫폼 다운 시: 강사 화면 공유로 시연하고 정제 프롬프트를 단톡방으로 배포)
7. 계정 사전 발급 + 종이 배부표 ([[무브-2-대량-계정-발급-키트]])
