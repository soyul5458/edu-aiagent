# 무브 2 — 대량 계정 발급 키트

## 왜

- 30명 수업 기준, 관리자 화면에서 계정 30개를 만들면 클릭 약 150회 + 15분 + 오타 리스크. 이 비용은 **수업마다 반복**된다.
- 스크립트 1회 구현(약한 모델로 1시간)으로 수업당 15분과 오타 리스크를 영구 제거하고, 종이 배부표까지 자동으로 나온다.
- 강사의 시간이 최대 병목이므로, 반복 작업 제거는 곧바로 수익률로 환산된다.

## 정확한 단계

1. 아래 프롬프트를 저렴한 모델(Claude Code)에 그대로 붙여넣어 `scripts/create-accounts.mjs` 생성
2. `names.txt`에 수강생 이름을 한 줄씩 입력
3. `node scripts/create-accounts.mjs` 실행 → `handout.csv` 인쇄
4. 수업 시작 때 배부표를 나눠주고, 종료 후 다음 기수 전에 이전 계정 일괄 정리 여부 결정

## 완료 기준 (done)

- [ ] names.txt에 10명 입력 → 실행 → 계정 10개 + handout.csv 생성
- [ ] 그중 1개 계정으로 실제 로그인 성공
- [ ] 관리자 페이지 계정 목록 수가 10 증가

## 약한 모델에게 시킬 프롬프트

```
edu_AIagent 프로젝트에 scripts/create-accounts.mjs 파일을 만들어줘.

동작:
1. 프로젝트 루트의 names.txt(한 줄에 이름 하나, UTF-8)를 읽는다
2. 각 이름에 대해 아이디를 student01, student02... 형식으로 만든다.
   기존 DB에 같은 아이디가 있으면 번호를 이어서 사용한다.
3. 비밀번호는 기억하기 쉬운 "영단어+숫자2자리" 6~8자로 무작위 생성한다 (예: apple37)
4. node:sqlite 의 DatabaseSync 로 data/platform.db 를 열어 users 테이블에 삽입한다.
   - 스키마: username TEXT UNIQUE, password_hash TEXT, name TEXT, role TEXT('student')
   - 비밀번호는 반드시 bcryptjs.hashSync(pw, 10) 으로 해시 (bcryptjs 설치돼 있음)
5. 완료 후 handout.csv (이름,아이디,비밀번호) 를 프로젝트 루트에 만들고,
   생성된 계정 수를 콘솔에 출력한다.

검증: 실행 후 스크립트가 DB에서 방금 만든 계정 수를 SELECT COUNT 로 다시 세어
콘솔에 붙여넣어야 완료다. 말로만 "완료"라고 하지 마라.
```
