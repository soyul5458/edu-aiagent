# node:sqlite 행은 클라이언트 컴포넌트에 그대로 못 넘긴다

## 문제

갤러리 페이지가 데이터가 생기자마자 서버 오류(`__next_error__`)로 빈 화면. 데이터가 없을 땐 멀쩡했다.

## 헛짚은 것

렌더링 코드나 SQL을 의심했지만, 둘 다 정상이었다. 데이터 "유무"에 따라 갈리는 게 단서였다.

## 해결

node:sqlite의 `.get()/.all()`이 반환하는 행은 **null-프로토타입 객체**인데, Next.js는 서버 → 클라이언트 컴포넌트 직렬화에서 이를 거부한다. `lib/db.ts`에 `toPlain()/toPlainOne()`(스프레드로 일반 객체화)을 만들어 모든 조회 지점에 적용.

## 다음에 쓸 규칙

- DB 결과를 클라이언트 컴포넌트 props로 넘길 땐 반드시 toPlain 계열 통과
- "데이터가 생기니 깨진다" = 데이터의 형태(직렬화)를 의심하라. 코드가 아니라 경계를 봐라
