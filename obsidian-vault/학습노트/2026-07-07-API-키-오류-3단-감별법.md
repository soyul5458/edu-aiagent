# API 키 오류는 상태 코드 조합으로 감별한다

## 문제

Fireworks 키를 넣었는데 계속 인증 실패. 키가 문제인지, 설정이 문제인지, 코드가 문제인지 구분이 안 됐다.

## 헛짚은 것

"키가 없다"와 "키가 틀렸다"를 같은 문제로 취급했다. 실제로는 3가지 서로 다른 상태다.

## 해결

상태 코드 조합으로 감별했다:
- 클라이언트가 "자격 증명을 찾을 수 없음" 예외 → 키가 **로드조차 안 됨** (env 변수명·서버 재시작 확인)
- 실존 모델 호출에 401 → 키가 로드됐지만 **서버가 거부** (키 값 자체가 틀림)
- 존재하지 않는 모델에 404, 실존 모델에 401 → 이 조합으로 "모델 문제 vs 키 문제"를 분리
최종 원인은 한 글자: 키 접두사가 `fw-`(하이픈)로 잘못 복사됨. 올바른 형식은 `fw_`(언더스코어).

## 다음에 쓸 규칙

- 인증 오류는 먼저 "로드 실패 vs 거부"를 나눠라. 고치는 곳이 다르다 (설정 파일 vs 키 발급처)
- 키·토큰은 눈으로 비교하지 말고 형식(접두사·길이)부터 확인하라. 한 글자 차이는 눈이 못 잡는다
- 구버전 서버가 살아있으면 어떤 수정도 "반영 안 된 것처럼" 보인다. 검증 전 항상 포트 기준으로 프로세스 정리
