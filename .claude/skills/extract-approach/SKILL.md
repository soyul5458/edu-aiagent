# extract-approach — 풀이 과정을 노트로 남기기

목적: 해답 코드가 아니라 **다음에 재사용할 판단 규칙**을 기록한다. 이 노트가 있으면
앞으로는 덜 똑똑한 모델도 같은 문제를 같은 수준으로 풀 수 있다.

## 절차

1. 이번 세션에서 해결한 사소하지 않은 문제를 고른다 (버그, 설계 결정, 막혔던 지점).
   여러 개라면 문제당 노트 1개씩 — 노트 하나에 통찰 하나.
2. `obsidian-vault/학습노트/YYYY-MM-DD-<주제>.md` 파일을 만든다. 형식:

```markdown
---
날짜: YYYY-MM-DD
태그: [학습]
---

# <제목: 결론을 담은 한 문장>

## 문제
(증상과 상황, 2~3줄)

## 헛짚은 것
(처음 시도가 왜 틀렸는지 — 같은 함정을 다시 피하기 위한 기록)

## 해결
(실제로 통한 접근, 핵심만)

## 다음에 쓸 규칙
(일반화된 판단 규칙 1~2줄 — 이 섹션이 가장 중요하다)

[[홈]]
```

3. `obsidian-vault/홈.md`의 "학습노트" 목록에 새 노트 링크를 추가한다.
4. 사용자에게 한 줄로 보고한다: 어떤 노트를 남겼는지.

## 하지 않는 것

- 세션 전체 요약 금지 — 통찰만 원자화한다 (긴 보고서는 잊힌다)
- 코드 전문 복사 금지 — 규칙과 이유만. 코드는 저장소에 이미 있다
